代码优雅设计

发表于 2018-05-22

代码设计

  • ADTS
    • 隐藏实现细节
    • 改动不会影响到整个程序
    • 更容易提高性能
    • 让接口提供更多的信息
    • 易读性
  • 一个类可以存在多个ADT,但是要保证他们在同一个抽象层次。
  • 提供成对的服务。
  • 把不相关的信息转移到其他的类中。 如果一个类中一半以上都是描述其他的事情，可以把这个类抽象出来。
  • 良好的封装
    • 尽可能限制类和成员的访问权限
    • 不要暴露成员数据
    • 不要对类的使用者做出任何假设。
  • 继承的类的方法必须具有相同的语义。

子程序的优雅设计

  • 名称的可读性
  • 输入的变量尽量为Const 避免操作内存产生一系列的问题
  • 防御式编程。
  • 参数不能过多。
  • 方法名称Cal,Print,
  • 关注编译器给的类型不匹配的警告。
  • 检查所有可能的返回路径。编写函数的开始设置一个缺省值，最后返回是一种很好的做法，这种做法能够在未正确设置返回值的时候提供一个保险网。
  • 宏程序 外层加大括号，内层多加小括号
  • 除非必要否则少用宏代替子程序，宏命名的可读性较差
